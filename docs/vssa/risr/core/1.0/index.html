RISR Core Vocabulary 1.0

Lightweight OWL2 vocabulary for RISR core semantics. Aligns with SEMIC/ISA² principles: minimal OWL constraints, SHACL for validation, reuse of SKOS + DCTERMS, bilingual LT/EN labels.

Conceptual data model (UML)

The conceptual data model defines the core entities and relationships underlying the RISR core vocabulary.

Model summary

ElementDescription
risr:EntityBase abstraction with technical metadata (_id, _revision, _txn, …).
risr:EnumEnumeration represented as skos:ConceptScheme.
risr:EnumItemEnumeration item represented as skos:Concept and also risr:Entity.
risr:codeBusiness identifier (subproperty of skos:notation).
risr:inEnumLinks an EnumItem to its Enum (intended 1..1; enforced in SHACL).

Model vocabulary

Name (LT) Name (EN) RDF property Range Cardinality Definition Scope / Usage
Pavadinimas Label rdfs:label rdfs:Literal 0..1 Human-readable name of the object. Used for displaying the object name to users.
IRI (nuoroda) IRI (identifier) — IRI 1..1 Canonical identifier of the resource. Used as the unique RDF identifier of the resource.
Intervalas Range rdfs:range rdfs:Resource 0..1 Defines the range of values that a property can take. Used in property definitions.
Kardinalumas Cardinality owl:cardinality xsd:nonNegativeInteger 0..1 Specifies the allowed number of values for a property. Used to constrain data in SHACL / OWL.
Apibrėžimas Definition skos:definition rdf:langString 0..1 Formal textual definition of the term. Normative description of the meaning of the term.
Panaudojimas Scope skos:scopeNote rdf:langString 0..1 Notes on how and when the term should be used. Provides guidance on practical usage.
_type Type rdf:type rdfs:Class 1..1 Code name of the model or namespace. Corresponds to the DSA model name.
_id Identifier dct:identifier xsd:string 1..1 Unique object identifier in UUID format. Used to uniquely identify an object.
_revision Revision dct:identifier xsd:string 1..1 Object version identifier (revision number). Used to ensure data integrity during updates. Each modification requires matching revision numbers; the revision changes whenever the object is updated.
_txn Transaction ID dct:identifier xsd:string 1..1 Transaction identifier. During a single request, multiple objects may be modified; all such objects share the same transaction identifier.
_data Data list rdf:List rdfs:Resource 1..1 List of objects when multiple objects are returned together. Used when returning multiple objects in a container.
_cid Sequence ID rdf:subject xsd:string 1..1 Sequence identifier. Used as a change identifier in changes queries.
_created Created date dct:created xsd:dateTime 0..1 Date and time when the record was created. ISO 8601 date-time format with timezone: YYYY-MM-DDThh:mm:ss±hh:mm.
_updated Updated date dct:modified xsd:dateTime 0..1 Date and time when the record was last modified. Used in changes queries; ISO 8601 format with timezone.
_label Human-readable label rdfs:label rdfs:Literal 0..1 Human-readable name of the object. Intended for display purposes.
